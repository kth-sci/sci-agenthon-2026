# SCI Agentathon 2026 — Track C

**Open Agent Exploration: KTH systems, knowledge work, and lab automation**

**Lead:** Wei Ouyang (wei.ouyang@scilifelab.se)  
**Room:** Hållfasthetslära seminarierum, Teknikringen 8D, floor 1

## What is Track C?

Track C is the free-exploration track. There is no fixed tutorial or prescribed problem set. Wei will open with a starter set of agent tools, then each sub-team picks what they want to build and runs with it for the day.

## Suggested directions

Pick any direction that is relevant to your own research or administrative work. Past participants have found the following areas particularly productive:

- **Administrative workflow agents** — automate repetitive KTH processes such as eISP form filling, scheduling coordination, or document review pipelines.
- **Retrieval-augmented knowledge agents** — build a RAG agent over your group's papers, reports, or internal documentation so the agent can answer questions grounded in your actual material.
- **Lab automation agents** — connect an agent to lab instruments, data pipelines, or LIMS systems to run or monitor experiments hands-free.
- **Document generation and review** — draft reports, grant sections, or slide decks from structured data; have a second agent review and iterate.
- **Custom KTH system integrations** — wrap any KTH API or web interface with Claude for Chrome or Cowork-mode connectors.

You are not limited to the above. If you have an idea that does not fit here, bring it — Track C is the right place for it.

## How the day works

1. **10:15–10:30** — Hands-on starter: Wei demonstrates a few agent patterns you can build on.
2. **10:30–12:00** — Coding session 1: your sub-team forms, picks a direction, and ships a first working prototype.
3. **13:15–15:45** — Coding session 2: iterate toward a demoable result.
4. **16:00–16:45** — Each sub-team delivers a short live demo in Salongen.

## Folder layout

Create a subfolder here for your sub-team and work inside it:

```
Track_C/

  <your-team-name>/  # create this — your working directory for the day
    scratch/         # private scratch space (ignored at deliverable collection)
```
