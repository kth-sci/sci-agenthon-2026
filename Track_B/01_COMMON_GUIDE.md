# Common Guide

Use this short guide across all three sub-teams.

## Working Pattern

The core loop is:

```text
inspect -> plan -> approve -> edit -> verify -> report
```

Before the agent edits files, ask it to explain what it saw and propose the smallest useful change. After any edit, ask for the actual diff and rerun the relevant command from <a href="02_COMMANDS_AND_VERIFY.md" target="_blank" rel="noopener noreferrer">02_COMMANDS_AND_VERIFY.md</a>.

## Role File Setup

Each sub-team has a role template in `agent_role_templates/`. During Task 1, copy the matching role template into the sub-team folder and rename the copy:

- `CLAUDE.md` if you are using Claude Code;
- `AGENT_ROLE.md` for another agent tool;
- or paste the contents into your tool's system prompt or rules field if that is how the tool works.

The copied role file is the source of authority for the agent. After copying it, scope the agent tool to the specific sub-team folder only:

- `Track_B/01_hpc_simulation_workflow`
- `Track_B/02_research_code_debugging`
- `Track_B/03_scientific_postprocessing`

Then ask the agent to read the copied role file before it inspects or edits anything. The role file and the folder scope work together: one tells the agent what it may do, and the other limits what it can touch.

## Optional Team Roles

Roles can help a group collaborate, but they are suggestions. Use the ones that help and ignore the rest.

- **Driver:** controls the terminal and editor.
- **Agent wrangler:** writes prompts and keeps the agent scoped.
- **Scientific skeptic:** asks what evidence supports each claim.
- **Reporter:** updates the demo report as work progresses.
- **Demo lead:** shapes the final 3-5 minute story.

## Optional Progress Check

If the group wants a lightweight checkpoint, write five lines in the demo report:

```text
Verified so far:
Current blocker:
Next check:
Files changed:
One thing we will not overclaim:
```

This is an aid, not a schedule.

## Good Demo Principles

A strong demo can be small. It should show:

- the original problem or suspicious behavior;
- how the agent was constrained;
- one change or artifact the group created;
- one command that regenerated or verified it;
- one limitation or assumption that remains.

## Common Pitfalls

- If the agent edits before explaining, ask it to stop and give evidence first.
- If the agent treats a local run as cluster proof, ask what would require real cluster access.
- If the agent changes tests to make them pass, ask whether the test encodes a scientific expectation.
- If the agent makes a claim from a plot, ask for the supporting value, table, or figure.
- If a report cannot be regenerated by a command, treat it as a draft.

## Prompt Starter

Use your own words if you can. A good first message is:

```text
Please read the copied role file and the sub-team guide first. You are scoped to this sub-team folder only. Summarize your role, scope, restrictions, and the first suggested step. Do not edit files until we approve a plan.
```
