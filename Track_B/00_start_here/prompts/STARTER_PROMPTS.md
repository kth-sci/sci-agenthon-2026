# Starter prompts (optional examples)

These are **optional** example phrasings for kickoff, to use **after** you have set up
your agent per [`../README.md`](../README.md) (copied a role template into your
sub-team folder, scoped the agent, asked it to read the role document).

Try communicating with the agent **in your own words first**. Reach for these only if
you want a model phrasing. The **role document is the source of authority**, not this
file — and do not paste anything blindly if it does not match your situation.

## Shared opener — confirm the agent knows its role

Your first message should confirm the agent has actually read and understood its role
document before any work begins.

> *Example:* I have already (a) copied the role template into my sub-team folder as
> `CLAUDE.md` (Claude Code) or `AGENT_ROLE.md` (any other tool), (b) scoped this tool
> so the agent can access only that sub-team folder, and (c) asked you to read that
> role document. Please summarise back your role, scope, restrictions, and workflow
> before doing anything else.

## Sub-team 01 — HPC simulation workflow

Your message should point the agent at your README, ask it to summarise the Level 1
goal in its own words, and propose a first concrete step without editing files yet.

> *Example:* My sub-team is `01_hpc_simulation_workflow`. Our mission is to build an
> assistant that helps a researcher set up, launch, and monitor a harmonic-oscillator
> simulation sweep on a generic HPC-style workflow. Please read
> `Track_B/01_hpc_simulation_workflow/README.md`, summarise the Level 1 goal back to
> me in your own words, and propose the first concrete step. Do not edit any files
> until I confirm the plan.
