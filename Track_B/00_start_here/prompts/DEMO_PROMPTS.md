# Demo prompts (optional examples)

You should already have been updating your sub-team's `DEMO_REPORT_TEMPLATE.md` after
each task throughout the day. These are **optional** example phrasings for pulling that
running report together into your final readout. Your agent should already be scoped to
your sub-team folder and operating off its role document (see [`../README.md`](../README.md)).

Try asking in your own words first. Use these only if you want a model phrasing — the
**role document is the source of authority**, not this file. Use only what you actually
built today; no aspirational claims.

## Sub-team 01 — HPC simulation workflow

Your message should ask the agent to help fill in your sub-team's demo report from what
you actually built, then draft a short talking-points outline and suggest one thing
worth showing live.

> *Example:* Please help me fill in
> `Track_B/01_hpc_simulation_workflow/DEMO_REPORT_TEMPLATE.md`, saved as
> `Track_B/01_hpc_simulation_workflow/DEMO_REPORT.md`, using only what we actually
> built today. Then draft a 3-minute talking-points outline (what we built, what
> works, what is next) and suggest one command or screenshot worth showing live.
