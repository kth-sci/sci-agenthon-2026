# Track B Demo Rubric

Use this for coaching, not grading. A strong demo is honest and reproducible, even if the artifact is small.

| Criterion | 0 | 1 | 2 |
|---|---|---|---|
| Problem framing | Vague task | Concrete task but weak research context | Clear research workflow problem |
| Agent control | Agent acted freely | Some scoping or approval | Scope, plan, approval loop clearly shown |
| Artifact | No reusable output | Output exists but hard to rerun | Reusable script, test, check, report, or figure |
| Verification | "It seems to work" | One command or visual check | Explicit command, result, and evidence shown |
| Scientific guardrail | None | General plausibility | Test, check, or metric tied to physics, data, logs, or workflow assumptions |
| Limitations | None stated | Generic caveat | Specific verified vs unverified boundary |
| Transfer | Not discussed | Weak analogy | Clear next use in real SCI research |

## Minimum Demo Script

```text
1. The original problem was ...
2. We constrained the agent by ...
3. The agent helped us change or create ...
4. We verified it by running ...
5. The result shows ...
6. We should not overclaim ...
7. In our own research, this pattern maps to ...
```

## Time Target

Three minutes per sub-team if many teams present; five minutes only if there are very few sub-teams.
