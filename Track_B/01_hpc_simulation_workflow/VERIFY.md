# Setup and verification commands

This file lists the exact commands for setting up a virtual environment and running the
verification steps the README asks for. Where the command differs between Windows
PowerShell and Linux/macOS, both are shown. Nothing here submits to a cluster.

## 1. One-time environment setup

### Windows (PowerShell)

```
cd C:\agentathon-track-b\Track_B
python -m venv .venv
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### Linux / macOS

```
cd Track_B
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### Quick dependency check (both platforms)

```
python -c "import numpy, pandas, matplotlib"
```

If this exits silently, you are set. If it raises `ModuleNotFoundError`, repeat the
setup step above with the environment active.

## 2. Run the verification commands

With the environment active, move into this sub-team folder and run the checks.

### Windows (PowerShell)

```
cd .\01_hpc_simulation_workflow
python scripts/validate_package.py
python scripts/run_sweep_local.py --dry-run
```

### Linux / macOS

```
cd 01_hpc_simulation_workflow
python scripts/validate_package.py
python scripts/run_sweep_local.py --dry-run
```

The full local sweep (`python scripts/run_sweep_local.py`, no `--dry-run`) is part of
Task 3 — it writes under `outputs/`, which is gitignored. Do not commit anything in
`outputs/`.

## Adding your own checks

From Task 4 onward you will extend `validate_package.py` and may add scripts or a final
product. Treat every new artefact as something that must be **regenerable and verified**:

- a new check should be runnable as part of `python scripts/validate_package.py`;
- a new script or report should have one command that regenerates it;
- prefer defining the expected output and the check **before** you implement.

## What to expect

- **Initial state (as shipped):** `python scripts/validate_package.py` prints a
  PASS / WARN / FAIL report and exits nonzero if anything fails — it gives you useful
  information and may point at issues to repair. `python scripts/run_sweep_local.py
  --dry-run` prints the per-run plan and writes nothing.
- **After Task 3:** the full local sweep runs and produces `outputs/local_sweep_summary.csv`
  and `outputs/local_sweep_diagnostics.png`.
- **After Task 4:** validation catches or rules out common pre-submission problems and
  reports them clearly.
- **After Task 5:** your final product (report, runbook, diagnosis tool, etc.) can be
  regenerated by a single command.
