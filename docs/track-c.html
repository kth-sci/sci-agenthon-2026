What is Track C?

There is no tutorial, no fixed problem set, no prescribed path. Track C is a full day of free exploration. Wei will open with a crash course on Claude Code and a starter set of agent tools, then you take it wherever you want.

If the problem you want to solve isn't covered by Track A (teaching) or Track B (cluster/HPC), it belongs in Track C. Administrative workflows, knowledge retrieval, productivity tools, lab automation, document generation — if it involves an agent and it's relevant to your work at KTH, it's fair game.

The goal is simple: co-create a mini-project during the day and give a live demo at 16:00. Wei will be in the room all day to help you get unstuck, suggest directions, and push toward something demoable.

Form a Sub-Team

Work in groups of 2–3 people. Use Git for collaboration so everyone can contribute and the work is preserved after the event.

👥

2–3 people

Small enough to move fast. Large enough to cover different skills — one person to prompt, one to review, one to steer.

🌿

One Git repo

Create a repo on GitHub or use the shared OneDrive. Claude Code works best inside a git-tracked directory — you get a full undo history.

🎯

One demo

Aim for a single, focussed deliverable that you can show live in ~10 minutes at 16:00. A narrow thing that works beats a broad thing that doesn't.

Git quick-start for your team

mkdir my-agentathon-project && cd my-agentathon-project
git init && claude # start Claude Code, run /init
# push to GitHub and add teammates as collaborators
# teammates: git clone <url> then open with claude

Topic Ideas

Not exhaustive — these are starting points. Your own research or daily frustrations are better inspiration than this list.

KTH Admin

Administrative workflow agents

Automate KTH IT processes — eISP individual study plan submissions, scheduling coordination, committee review workflows, report generation. Claude for Chrome can interact with web forms directly.

Knowledge

Retrieval-augmented knowledge agents

Build a RAG agent over your group's papers, internal reports, thesis collection, or KTH wiki content. Ask questions grounded in your actual material — not generic AI answers.

Lab

Lab automation agents

Connect an agent to lab instruments, data pipelines, or LIMS. Run or monitor experiments with natural language commands. Schedule protocols, parse results, trigger follow-up measurements.

Writing

Document generation and review

Draft grant sections, project reports, slide decks, or tenure documentation from structured data. Have a second agent review for clarity, consistency, and completeness.

Integrations

KTH system integrations

Wrap any KTH web interface with Claude for Chrome or build a custom skill. Canvas course data, publication databases, conference systems, ordering systems — anything with a web UI or API.

Your idea

Anything not covered by Track A or B

If your daily frustration isn't teaching-related (Track A) or cluster-related (Track B), bring it here. The best demos come from real problems, not examples from a list.

Getting Started

1

Create a folder and start Claude Code

mkdir my-project && cd my-project && git init && claude

Once inside Claude Code, run /init to generate a CLAUDE.md from your project structure.

2

Tell Claude what you want to build

Describe your goal in plain language. Don't over-specify the steps — describe the outcome. Claude decides how to get there; your job is to review the result and apply domain judgment.

3

Iterate toward something demoable

Small follow-ups compound. "Make it handle the edge case where the field is empty." "Add a summary at the end." Keep refining the same thread rather than restarting.

4

Demo it at 16:00

One live demo per sub-team. ~10 minutes. What did you build? Show it running. What surprised you? What would you do next?
