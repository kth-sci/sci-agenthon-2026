The grading problem

Handwritten physics

Closed rubric

Swedish feedback

Workflow inputs
03 / 12

The agentic workflow used five local input folders with distinct roles.

Exam/

The official exam PDF. Used as the source of problem statements, numbering, and expected exam structure.

Solution/

Teacher solution PDF. Used to understand intended physics reasoning and correct final results.

Grading_rules/

The grading rubric PDF. Converted into the closed JSON rubric used by the grader.

Human_graded/

Teacher-annotated PDFs and XLSX grades. Used only as ground truth for evaluation and comparison.

Ungraded/

Raw student submissions. These are the PDFs the agent grades problem-by-problem.

Privacy-sensitive source material stays local and remains excluded from git.

Phase 1 shipped
05 / 12

A reproducible grader, not just a prompt.

The demo packaged the grading behavior as a project skill plus a stdlib-only evaluation harness that joins agent output to human grades.

  • Rubric JSON validator: all 8 problems sum to 1.0 p.
  • Page indexing pass before grading begins.
  • Structured JSONL output per student and problem.
  • Eval script computes letter grades and comparison tables.
  • Post-review hardening committed after the demo run.
Core artifacts: rubric, skill, eval module, comparison report.
How it runs
06 / 12

The agent is a small pipeline with explicit checkpoints.

1

Student PDF

One handwritten submission under Ungraded/.

2

Page index

Locate where each problem appears before scoring.

3

Rubric loop

Grade each problem against exact item and special-case rules.

4

JSONL row

Persist points, evidence, comments, and page references.

5

Eval report

Join by canvas ID and compare to teacher grades.

Letter grades are computed by eval, not by the agent.

Pitch to colleagues
12 / 12

The next phase is calibration and reuse.

Phase 1 proves the workflow can run end-to-end. Phase 2 should make it exam-agnostic and safer around decisions that matter most.

1

Keep the structured rubric loop

It makes grading inspectable, reproducible, and easy to compare against human ground truth.

2

Add threshold-aware caution

Near pass/fail and B-del thresholds, require stricter evidence or route to human review.

3

Parameterize for future exams

Generalize over rubric JSON, exam metadata, page indexing, and comparison files.

Bottom line: promising as an examiner assistant, not a standalone grader yet.
